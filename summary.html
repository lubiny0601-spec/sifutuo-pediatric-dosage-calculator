临床智能化剂量计算器生成与发布标准作业程序 (SOP)

文件编号： SOP-CLN-DOS-001
制定团队： 抗菌药物计算器开发团队
生效日期： 2026年7月11日
适用范围： 临床医护工具快速开发与部署

1. 目的与背景

为了规范并加速临床抗感染药物剂量计算辅助工具的开发，防范人工换算失误导致的用药差错，确保计算规则可追溯、算法可验证、界面高可用，特制定本标准作业程序（SOP）。本SOP完全聚焦于 Web H5 原生网页形式的轻量化项目，确保支持多药品独立链接发布和离线使用。

2. 核心操作规范（分步实施）

第一步 官方说明书数据提取与评估 (Extraction)

接收到目标药物的官方说明书后，提取以下六大核心指标并填入规则库模板：

  • 患者分组边界： 明确是否包含早产儿（纠正胎龄 PMA）、足月儿、儿童和成人。
  • 基准剂量算法： 按体重（mg/kg）、体表面积（mg/m²）或成人固定剂量（mg）计算。
  • 安全拦截限制： 提炼绝对禁忌症（如特定年龄以下肾损禁用、特定肌酐清除率下禁止给药）。
  • 肾损害阶梯分级： 提取并对照说明书的 eCrCL 区间划分阀值。
  • 肝功能调整机制： 确定合并肝胆梗阻或严重肝损时的剂量上限限制。
第二步 计算规则库结构化编译 (Compilation)

将提取的数据编译为标准 rules.json 文件，并进行前后台的解耦管理：

  • 使用规则库模板配置药物的基础数据，避免将用药克数和拦截警示文案写死在 JavaScript 计算逻辑中。
  • 规则库结构需采用标准 JSON。
第三步 原生 JS 算法引擎开发 (Engine.js)

编写与前端 UI 无关的纯原生 JavaScript 计算引擎 engine.js，必须涵盖：

  • 数据验证： 严密校验体重、年龄、肾功能等输入的合法性，防止负数或超出常规范围的数据导致溢出。
  • 封顶限额截断： 实施保护性截断（如儿童单次剂量不得超过成人对应状态下的单次上限；肝肾联合损害时强制进行 Cefoperazone 每日最大 2g 的截断折算）。
  • 复溶抽吸换算： 引入稀释换算公式，自动输出医护人员配液时需从小瓶中抽取的实际体积（如舒普深按复溶总浓度 375 mg/mL 折算，抽吸体积 mL = 剂量 / 375）。
第四步 H5 前端交互设计与防缓存处理 (UI & Anti-Cache)

在前端页面 index.html 和 app.js 的构建中，需满足以下设计要求：

  • 拟物美学主题： 采用深色背景搭配高饱和度电光蓝和荧光青，融合高对比度文本卡片。
  • 输入组件动态联动： 成人模式下隐藏儿童特有控件并挂载 CG 计算器；儿童模式下勾选“≤7天新生儿”时自动锁定频次为 q12h 并禁用下拉选择。
  • 缓存穿透处理： 页面引用的脚本和 JSON 资源必须在链接后方追加当前时间戳参数，例如 src="app.js?v=timestamp"，以规避医院内局域网代理强缓存导致的规则更新滞后。
第五步 回归测试与质量控制 (Verification)

在发布前，必须在 tests/engine.test.js 中编写回归测试代码，并使用 Node.js 原生测试套件进行验证：

  • 测试场景至少涵盖：正常肾功能、各阶段肾损、血液透析、儿童常规范围、儿童严重感染、儿童剂量安全封顶截断、首周新生儿 q12h 强制联动等。
  • 在终端执行：node --test tests/engine.test.js
  • 确保控制台输出 pass 100% 且无警告后方可进行提交。
第六步 多项目静态部署与分发 (Deployment)

为满足多药品计算器并存且链接名字独立的需求，采用子目录分支化发布流程：

  1. 将各药品的完整 H5 文件（HTML、JS、rules）分别复制到对应的拼音子文件夹中，例如：
    • 思福妥® (头孢他啶阿维巴坦)：/sifutuo/
    • 思福诺® (头孢他啶)：/sifunuo/
    • 舒普深® (头孢哌酮钠舒巴坦钠 2:1)：/shupushen/
  2. 在仓库根目录下，编写用于药品选择和导航的仪表盘门户首页（index.html）。
  3. 将更改提交并推送到 GitHub 远程仓库的主分支 master。
  4. 进入 GitHub 仓库 Settings -> Pages，将 Branch 设为 master，路径选择为 /(root) 并保存。
  5. 一分钟后，即可通过子路径分享各自药品的计算器网址。

4. 常见问题 (FAQ)

  • Q: 为什么在院内微信扫描链接打不开或显示旧规则？
    A: 这是由于浏览器强缓存所致。SOP已将引用链接在 index.html 中添加了 ?v=timestamp 随机戳。如果依然有问题，请在微信中长按页面选择“刷新”即可清除缓存。
  • Q: 如何把计算结果发在微信沟通群或记入病历？
    A: 点击计算结果底部的“复制计算结果”按钮，系统会自动将格式化后的病历文本写入剪贴板，直接粘贴即可。
