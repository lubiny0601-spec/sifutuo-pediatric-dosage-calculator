思福妥® 儿童抗感染剂量计算工具
项目开发流程与技术复盘报告

一、 项目背景与业务痛点

在儿科抗感染临床用药中，注射用头孢他啶阿维巴坦钠（思福妥®）是重症感染的重要武器。然而，儿科用药剂量极度复杂，需要根据患儿的纠正胎龄（PMA）、日龄、月龄、以及肾清除率（eCrCL）等进行阶梯式剂量与频次微调，且存在严格的单次剂量上限封顶和安全拦截禁忌。

临床一线人员及代表在学术沟通中，翻阅说明书核算不仅效率低下，且在紧急状态下存在极大的核算风险。因此，构建一个“防错、秒级匹配、离线可用且尊重隐私”的剂量数字工具是核心诉求。

二、 核心技术栈选型与设计原则

项目在技术选型上拒绝了繁重的打包框架，采用了现代 Web 极简设计原则：

  • 核心引擎： 采用纯 Vanilla JavaScript 编写的 engine.js，同时支持 Node.js 环境（用于运行单元测试）和浏览器端（用于界面渲染），无需重复编写逻辑。
  • 编译机制： 使用 Node.js 的 xlsx 库开发了 compile.js，自动将 Excel 中配置的 20 行匹配规则转换为静态 JSON 库。代码和剂量规则解耦，未来规则更新只需修改 Excel 即可一键编译。
  • UI 交互细节：
    • 同心圆角（Concentric Radii）： 整体卡片采用 20px 圆角，内层输入表单采用 10px 圆角，视觉过渡自然。
    • 弹性手感（Tactile Feedback）： 所有按钮和选项卡均添加了 active:scale-[0.96] 的微下沉弹性过渡，增加触摸质感。
    • 数值等宽（Tabular Numbers）： 对计算结果运用 font-variant-numeric: tabular-nums 样式，防止数值计算波动时排版出现位移抖动。

三、 标准软件开发生命周期 (SDLC) 实践

本项目完整落地了现代敏捷软件工程的三大核心实践：

1. 规则矩阵标准化与校验

项目排除了“思福诺”相关干扰数据，全量录入思福妥说明书规则，设计了 5 档肾损系数（Normal, 31-50, 16-30, 6-15, ESRD），对应儿科特定的调节系数（中度 0.4，重度及透析 0.3）。

2. 测试驱动开发 (TDD) 验证

在编写计算逻辑前，预先设计了 8 个涵盖全边界测试场景的单元测试，包含大体重封顶、极低体重拦截、新生儿肾损拦截等，确保计算结果 100% 具备确定性。通过 node --test 实现了极速测试反馈。

3. 微信小程序与 H5 容器集成 (容器化部署)

利用 GitHub Pages 托管核心 H5 网页，微信小程序端仅作为 <web-view> 容器壳载入页面。这样不仅实现了小程序的即开即用，且将代码维护和发布收拢于 Git 仓库，推送即生效，免去了每次微调都需要微信官方漫长审核的痛点。

四、 核心架构亮点

亮点 1：Serverless 隐私合规设计
所有患儿的敏感生理参数（体重、年龄、肾功能）均在用户的手机本地浏览器完成本地化即时计算，数据完全不上传服务器。100% 保护患者隐私，极大降低了合规风险。

亮点 2：数据与逻辑彻底分离
规则以 Excel 为源头，自动编译成 JSON 文件。任何非技术人员均可通过修改 Excel 完成对剂量计算逻辑的调整，项目具备极高的生命力与可维护性。
